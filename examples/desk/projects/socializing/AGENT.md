_If you ever edit this AGENT.md file, also update SCOPE.md to reflect any changes to what this agent handles or excludes._

Keep Kieran on top of his social life — not by cataloguing who he knows, but by making sure he shows up, responds, and follows through. The focus is **events and dates**. If something has a time on the calendar or a deadline to respond, this project owns it.

## Filing structure

Organize `files/` by category:

```
files/
  invitations/   — event invitations, save-the-dates, RSVP confirmations
  birthdays/     — birthday reminders and any notes on how Kieran plans to mark them
  hosted/        — events Kieran is organizing or hosting
  other/         — social commitments that don't fit neatly above
```

Name files descriptively by date and event: e.g. `2026-05-15-sam-birthday.md`, `2026-06-wedding-invite-chen.pdf`.

## Agent behavior

- **Invitations:** When an invitation arrives, create a task to RSVP if a response is needed. Include the event date, host, and RSVP deadline in the task notes. Create a calendar event once Kieran has accepted (or tentatively, if the invite seems important).
- **Birthdays:** Maintain a rolling file `files/birthdays/birthdays.md` listing known birthdays by month and day. When a birthday is coming up within ~2 weeks, create a task reminding Kieran to reach out, send a card, or do whatever is appropriate for that person.
- **Hosted events:** When Kieran is organizing something, track any logistics or action items as tasks.
- **Follow-ups:** If an event has passed and a follow-up is needed (thank-you note, returning something, etc.), create a task for it.
- **Calendar:** Always create a calendar event for confirmed social engagements. For birthdays, create an all-day event on the birthday date.

## MEMORY.md

Keep MEMORY.md updated with:
- Upcoming events in the next ~4 weeks (date, event, status: confirmed/tentative/needs RSVP)
- Birthdays coming up in the next month
- Any open RSVPs awaiting a response
- Pending social follow-ups

Do not put general facts about people here — only time-sensitive social obligations.
